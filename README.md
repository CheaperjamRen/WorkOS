# WorkOS — AI 工作操作系统

**把 AI 从「答题机器」变成「你的多项目工作调度员」**

---

## 这是什么

WorkOS 是一套基于 Claude Code 的个人工作系统配置模板。

我是一个同时在推进多件事的人——白天有本职工作，晚上在探索出海副业，还有几个并行的工具项目在跑。每天我的脑子里都有 5 个项目的碎片同时存在：这个在等别人回复、那个卡在某个前置任务上、另一个新想法刚涌现出来……

我不需要 AI 帮我管理情绪，我需要 AI 帮我管理**这团乱**。

WorkOS 解决的核心问题是：**「我现在应该先做什么？」**

不只是告诉你任务清单，而是告诉你：
- 你现在有哪些项目在跑
- 哪些任务在等哪些任务（依赖关系）
- 哪两件事可以同时推进（并行机会）
- 在你信息过载、脑子转不动的时候，只告诉你一件事

## 和其他工具的区别

| | 普通 To-do App | mycc（个人成长 OS）| WorkOS |
|--|--|--|--|
| 输入形式 | 你自己整理好的任务 | 你主动描述的情绪和想法 | **任意混乱文字：聊天记录、AI 对话、会议纪要** |
| 核心输出 | 任务清单 | 情绪梳理 + 成长复盘 | **项目状态 + 依赖关系 + 并行机会** |
| 面向场景 | 任务执行 | 个人成长管理 | **多项目并行的知识工作者** |

## 核心功能

**🧠 脑倒（Brain Dump）**
把任意混乱文字丢进来——聊天记录、AI 对话总结、会议纪要、碎碎念全可以。AI 自动提取项目和任务，分配到 5 个池子，识别依赖关系，找出并行机会。

**📋 早报（Daily Brief）**
每天工作开始说「早报」。AI 读取你的项目状态，给出今日最优启动序列和并行机会，而不是把所有任务全部列出来让你焦虑。

**🗺️ 全局（Global View）**
说「全局」，看到所有项目的状态、谁在等谁、什么被阻塞、当前的解锁路径。

**📦 收工（Checkout）**
说「收工」，AI 自动更新项目状态，沉淀今天发现的规律，给出明日启动建议。每次收工，系统对你的工作方式多了解一点。

**三种工作模式**
- 正常模式：全局视角，适合普通工作日
- 🆘 蒙圈模式（「我现在脑子很乱」）：只给你一件事，其余屏蔽
- ⚡ 冲刺模式（「今天全力冲」）：最大化并行，极致效率编排

## 5 个任务池

所有任务只能进入这 5 个池子，不允许模糊地带：

| 池子 | 标准 | 典型例子 |
|------|------|---------|
| 🔴 今日必须推进 | 今天不做影响交付/别人/节奏 | 报告截止、等你回复的任务 |
| ⏳ 等待中/可并行 | 已发出，等别人回应 | 等代码跑、等同事确认 |
| 🔄 正在沉淀的流程 | 不急，但做了能省以后的时间 | 工作流自动化、工具搭建 |
| 💡 新产品/新想法 | 有价值，但今天不抢注意力 | 新功能 idea、新方向调研 |
| 🗄️ 暂缓/以后再看 | 有意思，但现在做会拖垮你 | 宏大计划、遥远的目标 |

## 快速开始

**第一步：把这个文件夹放到你电脑任意位置**

建议路径：`~/Documents/workos/`

**第二步：用 Claude Code 打开**

```bash
claude /你的路径/workos
```

**第三步：填写 `memory/work-profile.md`**

这是整个系统的底座。花 15-20 分钟认真填写——你的工作角色、同时在推进什么事、你的节奏偏好。越真实越好。

**第四步：第一次「脑倒」**

把你脑子里现在所有乱七八糟的事情全部丢进来。说「脑倒」然后开始粘贴或者直接说。这是你的项目状态初始化。

**第五步：养成两个习惯**

- 每天开始：说「早报」
- 每天结束：说「收工」

这两个习惯，让 WorkOS 越用越懂你。

## 设计原则（写给想理解的人）

> 为什么 WorkOS 这样设计，而不是那样设计

**配置 / 状态 / 记忆 三层分离**

- `work-profile.md`：静态，你是谁，几乎不变，你维护
- `project-state.md`：动态，现在在做什么，每天变，AI 维护
- `auto-memory.md`：积累性，AI 观察到的规律，只增不减

三者分离的好处：想更新你的工作角色，只改一个文件；项目状态有问题，只看一个文件；不会因为 AI 更新状态而意外覆盖你的档案。

**Skill 完全模块化**

每个 skill 文件包含：触发条件 + 执行步骤 + 输出格式 + 边界处理。删掉一个 skill，其他 skill 不受影响。新增一个 skill，只需新建文件 + 在 CLAUDE.md 触发词表加一行。

**CLAUDE.md 是单一入口**

Claude Code 启动时只读 CLAUDE.md。它引用其他文件，而不是复制内容。改行为只改一处。

## 维护成本

- **日常使用**：零维护。说「收工」，AI 自动更新状态
- **每季度**：花 15 分钟更新 `memory/work-profile.md`，让 AI 跟上你的变化
- **有重大变化**：直接告诉 AI，它会更新相应文件

## 如果你想定制

**如果你是程序员**：在 `skills/` 下新建 `code/skill-pr-review.md`，定义你的代码审查辅助流程。

**如果你做自媒体**：在 `skills/` 下新建 `content/skill-topic.md`，定义你的选题和发布流程。

**如果你做产品**：在 `skills/` 下新建 `product/skill-prd.md`，定义你的需求整理流程。

新 skill 不影响现有系统，插拔自由。
