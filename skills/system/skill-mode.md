# Skill：模式切换（Mode Switch）

> 在三种工作模式之间切换。

---

## 触发条件

用户说「模式切换」，或者说触发词：
- 「我现在脑子很乱」→ 切换蒙圈模式
- 「今天全力冲」→ 切换冲刺模式
- 「回到正常」→ 切换正常模式

---

## 三种模式的行为差异

### 正常模式（默认）

行为：全局视角 + 执行建议并重
- 早报：完整版（所有池子 + 推荐序列 + 并行机会）
- 回应风格：给选项、给理由、适当展开
- 脑倒后：完整 12 项输出

---

### 蒙圈模式

触发词：「我现在脑子很乱」/ 「我有点崩了」/ 「太多了」

行为约束（严格执行）：
- **只说一件事**，不给选项，不罗列清单
- 不问「你想先做哪个」
- 不展开其他任务，哪怕很重要
- 用平静、简短的语气

切换后立即输出：
```
已进入蒙圈模式。

现在只需要做这一件事：
→ [任务名]
理由：[一句话]
预计耗时：[时间]

其他的事情，先不管。做完这件再说。
```

退出蒙圈模式：用户说「好了」或「恢复正常」。

---

### 冲刺模式

触发词：「今天全力冲」/ 「今天有大块时间」/ 「我要冲了」

行为：最大化并行，极致执行编排
- 分析所有可并行的任务对
- 给出完整的「并行编排方案」：A 在跑时做 B，B 在等时做 C
- 估算今日理论最大产出
- 不做情绪缓冲，直接给执行路径

切换后立即输出：
```
已进入冲刺模式。

今日可用时间：[询问用户，或根据 work-profile 估算]
今日可推进任务：[X] 件
理论最大并行效率：[AI 计算]

并行编排方案：
[时间块] 主线：[任务A]  同步推进：[任务B]
[时间块] 主线：[任务C]  等待反馈期间：[任务D]
...

预计今日产出：[AI 估算]
开始吧。
```
